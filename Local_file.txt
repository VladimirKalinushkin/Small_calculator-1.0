Как установить Firefly III с помощью Docker

Firefly III можно установить с помощью Docker. Есть готовые контейнеры Docker практически для всех платформ. Если вы никогда раньше не использовали Docker, пожалуйста, сначала прочитайте его сами.

Самый простой способ установки Firefly III - это использование «Docker compose», который представляет собой инструмент Docker, позволяющий определять и запускать мультиконтейнерные приложения Docker. С сочинением Docker вы используете файл YAML для настройки сервисов вашего приложения. Затем, с помощью одной команды, вы создаете и запускаете все службы из вашей конфигурации.

Файл YAML и вся необходимая конфигурация предоставляются онлайн для вашего удобства.

Если у вас есть вопросы, пожалуйста, обратитесь к FAQ Docker или свяжитесь со мной, используя инструкции со страницы поддержки.
Docker Композитор

Скачайте файл Docker Compose и разместите его где-нибудь удобно. Чтобы включить импортера данных в вашу установку, ознакомьтесь с тем, как установить импортера данных.

Возьмите сырой файл и не вставляйте текст из своего браузера. Пробелы в файле очень важны. Так что используйте "Save As".
Скачать файлы конфигурации

Есть два файла конфигурации, которые вам нужны для запуска этого файла Docker Compose. Загрузите все файлы и сохраните их в той же папке, что и файл Docker Compose.

    Первый файл содержит переменные Firefly III и может быть загружен из репозитора Firefly III. Сохранить его как новый файл под названием .env.
    Второй файл содержит переменные базы данных и может быть загружен из хранилища Docker. Сохранить его как новый файл под названием .db.env.

Важно, чтобы вы переименовали файл в соответствии с инструкциями здесь. Вы можете увидеть в файле Docker compose, почему это так. Есть ссылка на это: env_file:. Если вы не называете его так, как он есть в файле Docker Compose, вы должны отредактировать файл Docker composite, чтобы соответствовать именам файлов.

Инструкции:

    Перемены DB_PASSWORDв .envКо всему другому. Выберите хороший пароль.
    Перемены MYSQL_PASSWORDв .db.envк тому же значению

Примечание

СНАЧАЛА сменить пароль. Если вы измените пароль после запуска Firefly III, он будет жаловаться на отсутствие доступа, потому что пароль хранился в объеме базы данных.
Перемены на PostgreSQL

Пользователи бегут arm/v7(некоторым пользователям Raspberry Pi), возможно, придется перейти на PostgreSQL, потому что на этих платформах недоступны надежные контейнеры под управлением MariaDB и MySQL. Для этого измените следующие строки в .envФайл.

DB_CONNECTION=pgsql
DB_HOST=db
DB_PORT=5432

Также изменить имена переменных в .db.env файл от MYSQL_* до POSTGRES_*. MYSQL_RANDOM_ROOT_PASSWORD можно удалить.

Изменить docker-compose.ymlфайл для указания postgresвместо того, чтобы mariadb:lts.

Затем изменить точку монтирования базы данных под томами (firefly_iii_db) для /var/lib/postgresql/data.
Запустите контейнер

Запустить следующую команду в каталоге, где оба docker-compose.ymlПрисутствуют все конфигурационные файлы.

docker compose -f docker-compose.yml up -d --pull=always

Вы можете следить за ходом установки, выполняя эту команду:

docker compose -f docker-compose.yml logs -f

Когда установка будет завершена, Firefly III поблагодарит вас за ее установку. Как только вы увидите это сообщение, вы можете посетить Firefly III. Он будет работать в вашем localhostместном хосте.
Серфинг для Firefly III

Теперь вы можете посетить Firefly III по адресу http://localhost или http://docker-ip:port, если он работает на пользовательском порту. Чтобы продолжить, прочитайте учебник о том, как создавать учетные записи и транзакции.
